# State and Artifacts

Knowledge_Service persists runtime data, published user-facing editions, and certification evidence in three top-level directories. None of these are Python packages; they are data roots referenced by path from the service root (`Path(__file__).resolve().parents[4]` in the morning runner).

## Overview

| Directory | Mutable at runtime | Git status | Primary consumers |
|-----------|-------------------|------------|-------------------|
| `state/` | Yes | Usually gitignored / local | Pipelines, collector, morning runner |
| `frontend/` | Yes (on publish) | Mixed (static assets + generated) | Browser, bookmark workflow |
| `runtime_evidence/` | Append-only snapshots | Committed certification bundles | Audits, bootstrap seed, docs |

---

## `state/` — live runtime store

`FileStateStore` (`intelligence/state.py`) reads and writes JSON/JSONL files under a single root. Production morning intelligence defaults to `<repo>/state/`.

### Bootstrap behavior

If `state/` exists but has no `profiles.json`, the morning runner either:

1. Copies all files from `runtime_evidence/phase512_optimization_20260701T074324Z/state/`, or
2. Seeds `profiles.json` from `config/profiles.json` when no bootstrap bundle is present.

### Top-level files

| File | Format | Purpose |
|------|--------|---------|
| `profiles.json` | JSON | Active intelligence watch profiles (corpus scope) |
| `episodes.json` | JSON | Discovered and processed podcast/web episodes |
| `source_graphs.json` | JSON | Per-source relationship graphs |
| `knowledge_objects.jsonl` | JSONL | Canonical KnowledgeObjects (chunks, embeddings metadata) |
| `growth_history.json` | JSON | Corpus growth metrics over time |
| `dedupe.json` | JSON | Deduplication fingerprints |
| `discovery_runs.json` | JSON | Discovery job history |
| `information_events.json` | JSON | New-information event log |
| `route_registry.json` | JSON | Runtime route certification state |
| `route_diagnostics.json` | JSON | Route health diagnostics |
| `certification_history.json` | JSON | Route/certification audit trail |
| `collector_config.json` | JSON | Collector tuning (e.g. acquisition delay) |
| `jobs.json` | JSON | Background job queue state |

### `state/analyst/` — Phase 4 analyst pipeline

| File | Purpose |
|------|---------|
| `claims.jsonl` | Extracted claims |
| `scored_claims.jsonl` | Relevance/novelty-scored claims |
| `corroboration_clusters.json` | Cross-source corroboration groups |
| `morning_briefs.json` | Legacy brief artifacts |
| `pipeline_runs.json` | Analyst run metadata |
| `synthesis/intelligence_items.json` | Synthesized intelligence items |
| `synthesis/themes.json` | Discovered themes |
| `synthesis/theme_history.jsonl` | Theme evolution timeline |
| `synthesis/intelligence_briefs.json` | Synthesis-layer briefs |
| `synthesis/pipeline_runs.json` | Synthesis run metadata |

### `state/production/` — Phase 5+ production layer

| File | Purpose |
|------|---------|
| `intelligence_briefs_v3.json` | Morning Brief v3 editions |
| `pipeline_runs.json` | Production pipeline run log |
| `morning_runs.json` | Daily morning intelligence summaries (append-only `runs` array) |
| `llm_cache.json` | Cached LLM brief enhancements (keyed by item + prompt version + model) |
| `llm_budget.json` | Persisted LLM usage accounting |
| `trend_history.jsonl` | Theme acceleration / trend history |
| `preferences.json` | Personalization preferences |

### Contract

- Writes use atomic replace (`.tmp` then `os.replace`).
- Invalid JSON raises explicit errors on read.
- State is **restart-persistent** but not transactional across files; pipelines write in stages.

## `frontend/` — published Morning Intelligence

Static site generated by `FrontendPublisher` (`production/morning/publisher.py`). No build step or dev server required.

### Stable entry points (bookmark these)

| Path | Description |
|------|-------------|
| `latest.html` | Primary daily edition (HTML shell + embedded data) |
| `latest.md` | Markdown edition for the current run |
| `data/latest.json` | Machine-readable brief + items + metadata |

The URL path never changes; content is overwritten each successful publish.

### Static assets

| Path | Role |
|------|------|
| `index.html` | HTML template copied into `latest.html` |
| `styles.css` | Stylesheet |
| `app.js` | Client-side rendering and “Copy AI Prompt” workflow |
| `scripts/prepare_frontend_data.py` | Optional data prep helper |

### Archive layout

Each publish may archive the prior edition:

```
frontend/archive/YYYY-MM-DD/
  morning.html
  morning.md
  morning.json
  run_summary.json
```

Archives provide historical editions without changing the `latest.*` bookmark.

### Publish flow

1. Morning runner builds `IntelligenceBriefV3` and markdown.
2. `FrontendPublisher.publish()` writes `latest.md`, `data/latest.json`, and regenerates `latest.html`.
3. Prior “Today” document may roll into `archive/`.
4. Run summary JSON is stored alongside archived editions.

## `runtime_evidence/` — certification snapshots

Immutable (by convention) directories capturing end-to-end certification runs. Used for documentation, regression proof, and **first-run state bootstrap**.

### Naming

```
<scenario>_<UTC-timestamp>/
```

Examples:

- `runtime_20260630T212721Z/` — foundation runtime certification
- `phase3_intelligence_20260701T002459Z/` — Phase 3 intelligence collection
- `acquisition_ladder_20260630T223321Z/` — transcript acquisition ladder
- `phase512_optimization_20260701T074324Z/` — production optimization (morning bootstrap source)

### Typical contents

| Subpath | Contents |
|---------|----------|
| `README.md` | Reproduce commands and artifact index |
| `EVIDENCE_MANIFEST.json` | File inventory and checksums |
| `raw/` | Inspector dumps, scheduler runs, blocker reports |
| `state/` | Point-in-time copy of `state/` (used for bootstrap) |
| `config/` | Profiles/routes at certification time |
| `reports/` | Human-readable certification reports |
| `transcripts/`, `segments/`, `knowledge_objects/` | Acquisition ladder artifacts |
| `logs/` | Command output and search traces |

### Relationship to live `state/`

- **Live `state/`** is the working database for daily runs.
- **`runtime_evidence/*/state/`** is a frozen snapshot for audits and optional seeding.
- Do not confuse evidence bundles with the active corpus; copying evidence into `state/` without intent resets production to that snapshot.

### Reproduce pattern

Evidence READMEs typically document:

```bash
export PYTHONPATH=src
./.venv/bin/python examples/<certification_script>.py
```

See individual `runtime_evidence/*/README.md` files for exact commands.

## Path resolution summary

| Component | Default path | Override |
|-----------|--------------|----------|
| State store | `<repo>/state` | `MorningIntelligenceRunner(state_dir=...)` |
| Frontend | `<repo>/frontend` | `MorningIntelligenceRunner(frontend_dir=...)` |
| Profiles | `<repo>/config/profiles.json` | Constructor `profiles_path` |
| Routes | `<repo>/data/source_routes.json` | Constructor `routes_path` |
| Bootstrap seed | `runtime_evidence/phase512_optimization_20260701T074324Z/state` | Hardcoded in `daily_runner.py` |
